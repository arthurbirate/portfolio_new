Object Detection with YOLOv5 & YOLOv8

A computer vision project focused on locating Waldo in complex, cluttered puzzle scenes using custom-trained YOLOv5 and YOLOv8 models. The project explores the challenge of small-object detection, compares model performance, and improves results using sliced inference with SAHI.

Objective

The goal of this project was to design and implement an AI system capable of automatically detecting and locating Waldo in highly cluttered puzzle images.

  • Handle heavy visual clutter and large numbers of distractors
  • Detect a very small target relative to the full image size
  • Distinguish Waldo from visually similar characters and patterns
  • Compare advanced real-time detection models for performance

The project applied YOLOv5 and YOLOv8 on a custom labeled dataset to learn Waldo’s visual characteristics and generalize to unseen scenes.

Dataset

Custom dataset creation, annotation, and augmentation strategy.

Data Preparation

I collected more than 200 high-resolution images from “Where’s Waldo?” books and online sources to create a dataset tailored to this problem. Each image was annotated in Roboflow using bounding boxes around Waldo to produce precise labels for supervised object detection.

To improve generalization and reduce overfitting, I applied data augmentation techniques such as flipping, blurring, and brightness adjustments. These transformations increased dataset diversity and helped the models perform better on unseen puzzle scenes.

Model Training

Comparative training approach for YOLOv5 and YOLOv8.

Training Workflow

I trained YOLOv5 and YOLOv8 separately to compare their accuracy and inference speed. Both models used transfer learning from pretrained weights to accelerate convergence and improve performance on a limited custom dataset.

  • Used pretrained weights for transfer learning
  • Tuned hyperparameters such as image size, batch size, and learning rate
  • Evaluated performance regularly during training to limit overfitting

This experimentation helped balance model precision, robustness, and inference efficiency.

Challenge of Small Object Detection

Why Waldo is difficult to detect and how sliced inference improved results.

SAHI Strategy

Waldo often occupies less than 1% of the total image area, which makes standard detection difficult. To address this, I used SAHI (Slicing Aided Hyper Inference), which splits large images into smaller overlapping tiles before inference.

Running detection on each tile separately improved recall by making Waldo appear larger relative to the tile, while also helping reduce false positives in cluttered scenes.

Results

Final performance and most effective model configuration.

Model Outcome

The YOLOv8 + SAHI combination delivered the best performance, achieving the strongest accuracy and clearly outperforming YOLOv5 in highly complex scenes. Even with sliced inference, the system remained fast enough for near real-time detection workflows.
