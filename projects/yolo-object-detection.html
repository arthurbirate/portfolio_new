Object Detection with YOLOv5 & YOLOv8: Where’s Waldo?

Model Training Documentation GitHub Repo

Objective

The objective of this project was to design and implement an artificial intelligence system that can automatically detect and locate Waldo in highly complex, cluttered images from the popular “Where’s Waldo?” puzzle books. Unlike traditional classification tasks, this problem involves object detection in a visually challenging setting.

  • High visual clutter: Each image contains hundreds of characters, objects, and patterns, many of which resemble Waldo’s red-and-white outfit.
  • Small target size: Waldo is typically a very small figure compared to the overall image, making detection more difficult.
  • Distractors: The presence of lookalike characters and distracting backgrounds increases the challenge for both humans and machine learning models.

The project’s goal was to overcome these difficulties by applying YOLOv5 and YOLOv8, two advanced deep learning models specialized in real-time object detection. The models were trained on a custom dataset of labeled Waldo images to teach the system to distinguish Waldo from other characters and accurately predict his position within new, unseen puzzle images.

Dataset

I collected over 200 high-resolution images from “Where’s Waldo?” books and various online sources to build a dataset tailored for this project. Each image was carefully annotated using Roboflow, where bounding boxes were drawn around Waldo to create precise labels for the detection model. To ensure reliable training and evaluation, the dataset was divided into training, validation, and testing sets.

To make the model more robust and generalizable, data augmentation techniques were applied. These included image flipping to simulate different orientations, blurring to mimic out-of-focus or low-quality images, and brightness adjustments to handle varying lighting conditions. This preprocessing strategy significantly increased the diversity of the dataset and reduced the risk of overfitting, preparing the model to perform effectively on new and unseen puzzle images.

Model Training

I trained two models, YOLOv5 and YOLOv8, separately to compare accuracy and inference speed.

  • Used transfer learning from pretrained weights.
  • Tuned hyperparameters (image size, batch size, learning rate).
  • Regular evaluation during training helped prevent overfitting.

This experimentation allowed me to balance model precision and performance.

Challenge of Small Object Detection

Waldo often occupied less than 1% of an image, making him difficult for standard object detectors. To address this, I used SAHI (Slicing Aided Hyper Inference), which slices large images into smaller overlapping tiles. Processing each tile separately improved both recall (finding Waldo) and precision (avoiding false positives).

Results

The YOLOv8 + SAHI combination achieved the highest accuracy (mAP) and significantly outperformed YOLOv5 in complex scenes. Even with slicing, inference remained fast enough for near real-time detection.

Key Learnings

  • Why slicing is essential for detecting small objects in large images.
  • How to fine-tune and evaluate deep learning models like YOLOv5 and YOLOv8.
  • Effective dataset annotation and augmentation with Roboflow.
  • Balancing speed and accuracy for real-world applications.
  • Applying object detection to fun but challenging tasks like “Where’s Waldo?”.
